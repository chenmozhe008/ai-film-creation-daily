# AI 创作信息日报 - 系统架构

## 项目定位

面向影视导演、编导、叙事创作者、AIGC 内容团队的 **AI 创作作战简报**。不是泛技术新闻聚合，而是服务创作决策的日报。

服务 4 类角色：
- **导演/编导**：叙事、分镜预演、案例参考
- **视觉/美术**：风格生成、角色一致性、视觉开发
- **后期/剪辑/声音**：配音、字幕、视频生成后处理
- **团队负责人/制片**：平台动向、成本变化、商业机会

## 栏目结构（4 个板块）

1. **行业与平台动向** - 大模型、平台能力、融资、政策、生态变化
2. **工具与能力更新** - 对创作链路有帮助的能力更新
3. **方法与经验** - 可执行、可迁移、可复用的创作方法
4. **作品/案例** - 值得拆解的作品与传播样本

## 时间窗口规则

- 固定：前一天 08:00 ~ 当天 08:00（Asia/Shanghai）
- 只收录 24h 内首发信息，宁缺毋滥
- 超过 36h 无条件不纳入
- 作品/案例板块放宽至 7 天内持续传播的热门作品

## 编辑原则

1. **事件级聚合**：同一事件多家报道只保留最权威一条
2. **服务创作不凑数**：每条至少满足 2 项价值标准
3. **分类稳定**：板块边界清晰，不混淆
4. **原名优先**：作品名/工具名优先原发布名

## 自动化流程

日报通过 [OpenClaw](https://github.com/nicepkg/openclaw) 自动驱动：
1. 定时触发采集（13+ 渠道并行）
2. AI 评分筛选（6 角色关联度评分）
3. 生成日报写入飞书文档
4. 更新飞书总入口索引
5. 同步推送到本 GitHub 仓库

## 数据源配置

所有数据源配置开源在 `sources/` 目录：

| 文件 | 说明 |
|------|------|
| `rss-feeds.json` | 18 个 RSS 官方源（OpenAI、Google、Anthropic、Runway 等） |
| `x-lists.json` | 107+ X/Twitter 账号（7 个分类） |
| `keywords.json` | 内容过滤关键词（中英文） |
| `kol-sources.json` | KOL 发现种子池 |
| `blacklist.txt` | 屏蔽来源列表 |
| `template.md` | 日报输出模板 |
